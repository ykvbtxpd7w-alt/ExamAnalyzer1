# ExamAnalyzer

ExamAnalyzer - курсова програмна система для генерації та аналізу екзаменаційних білетів. Програма допомагає сформувати комплект білетів з банку питань, збалансувати їх за складністю, переглянути результат перед збереженням і експортувати білети у PDF.

## Основна Ідея

Кожне питання має формальну складність:

```text
real_complexity = base_complexity * type_weight
```

Ваги типів питань:

```text
theory  = 1.0
problem = 1.3
proof   = 1.6
```

Складність білета обчислюється як середнє значення складностей усіх питань:

```text
difficulty_score = sum(real_complexity) / number_of_questions
```

Для сумісності з поточними банками питань система також підтримує категорії:

```text
Easy
Medium
Hard
```

Якщо у питанні немає `base_complexity` і `q_type`, програма автоматично використовує наближені значення для категорії складності.

## Можливості

- вибір предмета;
- вибір тем;
- налаштування кількості легких, середніх і важких питань;
- налаштування загальної кількості балів за білет;
- генерація комплекту білетів без повторення питань;
- автоматичний розподіл балів між питаннями пропорційно до складності;
- передпогляд згенерованих білетів;
- ручна заміна питання у білеті;
- фільтрація замін за темою та складністю;
- пошук альтернативного питання за назвою або темою;
- перерахунок балів і складності після заміни;
- експорт комплекту білетів у PDF;
- відкриття збереженого PDF безпосередньо з програми.

## Формат Даних

Основні банки питань лежать у папці `data`.

Приклад поточного формату питання:

```json
{
  "title": "Інтеграли: Питання №1 (Medium)",
  "category": "Medium",
  "points": 5,
  "usage_count": 0
}
```

Під час генерації програма доповнює питання службовими полями:

```text
topic
base_complexity
q_type
type_weight
real_complexity
points
```

Поле `points` у фінальному білеті не береться напряму з JSON. Воно перераховується так, щоб сума балів у кожному білеті дорівнювала значенню, яке задав користувач.

## Унікальність Питань

Програма **намагається не повторювати** питання в межах одного комплекту. Якщо в банку не вистачає нових питань потрібного рівня, вона **дозволяє повтор** — спочатку ті, що ще не використовувались, потім уже взяті з найменшою кількістю повторів.

Помилка виникає лише коли для якогось рівня складності в банку **немає жодного** питання (наприклад, обрано 3 важких, а в темах немає Hard).

При ручній заміні спочатку пропонуються питання, яких ще немає в комплекті; якщо таких немає — показуються вже використані.

## Передпогляд Та Заміна

Після генерації користувач бачить передпогляд комплекту:

- список білетів;
- питання вибраного білета;
- бали за кожне питання;
- тему, категорію та складність питання;
- загальну складність білета.

Будь-яке питання можна замінити. Для заміни доступні фільтри:

```text
Теми: Та сама / Обрані / Усі
Складність: Схожа / Та сама / Будь-яка
```

Система не пропонує питання, які вже використані в інших білетах комплекту.

## PDF

PDF формується після передпогляду. Якщо після збереження користувач змінює питання в білеті, кнопка відкриття старого PDF блокується, бо файл уже не відповідає актуальному комплекту.

## Історія генерацій

Після кожної успішної генерації програма додає запис у `data/session_history.json`. Повні білети в історію не зберігаються — лише параметри, за якими їх можна відтворити через `seed`.

У головному меню є кнопка **«Історія генерацій»**: список записів, деталі та **«Відтворити комплект»** за збереженим `seed`.

Приклад запису:

```json
{
  "created_at": "2026-05-24 18:42",
  "seed": "482913",
  "subject": "math_analysis",
  "topics": ["Інтеграли", "Похідні"],
  "recipe": {"Easy": 3, "Medium": 2, "Hard": 1},
  "tickets_count": 30,
  "total_points": 100,
  "bank_fingerprint": "a1b2c3d4e5f67890"
}
```

`bank_fingerprint` — короткий хеш файлу банку предмета на момент генерації. Якщо банк змінився, старий `seed` може дати інший комплект.

Legacy-приклади старих форматів даних лежать у `data/examples/`.

## Запуск

Встановити залежності:

```bash
pip install -r requirements.txt
```

```bash
python src/ui/app.py
```
